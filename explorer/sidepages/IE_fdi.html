A fixed width (300px) div, with margin-left: 200px
 

If the browser window is narrowed enough, so that the "liquid" container should become insufficient to contain the green div (with its margin left), then the green div drops below the float, even if no more room is available there then beside the float (it simply moves vertically.) Moreover the liquid container stops to shrink (or better: IE enlarges it to enclose the green div.)
In other words, when the browser window is narrowed so that the grey (width:80%) container should have a width less than 500px (margin-left + width of the green div) then IE/Win does two wrong things:

  • It moves vertically the green div below the float.
  • It enlarges the grey div to contain the green one.

More standard compliant browsers simply allow the fixed width green div to overflow the container and doesn'n move it.
