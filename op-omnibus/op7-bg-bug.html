This is a page to show a curious background bug in Opera7. It appears that if a background image is placed in the body element, and positioned at the bottom of that element either with 'bottom' or '100%', and the body is longer than the viewport, the BG image is pushed below the bottom edge of the viewport. This demo page is optimized for a screen at 800x600 pixels, in a 'free' version of Op7/win2k. Above is an <img> version of the complete image. The html element has a border to show the bug more clearly.

Because the content is slightly longer than the viewport (800x600), the red BG image is partially below the html element. Were the content longer, the BG image would be far below. There is also some partial image repeating evident, apparently caused by scrolling. Reloading cleans the image up, but does not correct the faulty position. Op6 does not have this difficulty.

This bug can be fixed by wrapping the content in a div, and applying the BG image to that div, rather than the body element.

Update: Yet another bug that has been fixed in Opera 7.5, calling a for a "Woohoo!".
