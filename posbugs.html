The demo

This one is based entirely on absolute positioning. div#container (thick brown border) is positioned using 'left' and 'top', and has equal width and height (80px). It contains two absolute boxes, (black and green borders) and a static div to show the content area, (grey box) and thus the padding area as well (light blue around grey box). Note: Without the fixed dimensions on div#container, browser behavior would vary wildly. See Absolutely Buggy II to view this behavior.

The green and black bordered boxes are positioned to opposite corners by applying '0px' on top/left and bottom/right respectively. They each enclose a small box positioned the same way. All boxes are dimensioned as squares, and Tan hack (for the IE5.x incorrect box model, see source) is applied to all the boxes.

The behaviors

It is assumed that earlier browsers will have 'difficulties' with this demo.

The mighty Gecko engine (Moz, N6/7) renders flawlessly. IE5/Mac appears identical to Moz. Opera 7 is also correct.

IE/Win is close, considering it's IE. But notice those vertically stretched small boxes? It appears that IE will not allow the 'height' of these absolute boxes to be less than the 'line-height'. However, this applies only to the big and little boxes. The middle sized boxes will strangely honor 'height:0px'.

However, such a bug will only rarely affect a page. Not so with Opera 6.

Oh, my aching back!

Please note that in Op6 the black and green bordered boxes are (as one unit) exactly the same size as the grey content area. (!) It would seem that Opera is rendering the demo by first placing the container without any padding or borders. Then the contained absolute boxes are placed accordingly. Finally, the specified padding and borders are applied to the container box, which stretches it down and right without affecting those contained boxes. The same goes for the little boxes.

This is a big problem (if you care about Op6). Don't assume you'll correct it with a wrapper div either, because the bug will infect that div as well.

The Fix

Don't use borders or padding on such a container. If you must, an absolutely positioned div may be positioned over (or under) the container, so as to provide 'substitute' borders and padding.

Happily, Opera 7 has fixed this problem, and now plays nice with absolute positioning. Bow thrice in the direction of Norway, please.

Whoops!

Those using Op7.1 may notice that the demo is missing its grey content area, supplied by a nested div BG. This was pointed out to me by Chris Hester. See this page for details on a brand new Opera bug!
